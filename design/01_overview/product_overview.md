# MAO 平台 — 产品概述与设计原则

> **版本**：V9.0-PROD | **更新日期**：2026-04

---

## 1. 产品愿景与定位

MAO 平台定位为赋能全业务线的**"AI 智能体编排底座"与"统一运营工作台"**。其核心理念是："业务逻辑归业务系统，AI 调度与可视化编排归 MAO 平台"。平台通过底层的标准化接入和低代码组装，在前端通过自然语言 (LUI) 彻底释放业务生产力。

平台分为两大端：**C 端 LUI 统一工作站 (User Workspace)** 面向日常运营人员，提供自然语言驱动的 AI 助理体验；**B 端 MAO 管理控制台 (Management Console)** 面向架构师与管理员，提供智能体工厂、全局编排画布、技能注册中心等核心管理功能。

### 1.1 核心设计原则

| 原则 | 说明 |
|---|---|
| **控制面与执行面分离** | Router 大脑只负责意图分发，执行引擎负责具体推演，两者物理隔离 |
| **卡片即技能** | 大模型绝不直接生成 UI 代码，而是输出 JSON 参数调用预设的"视图技能" |
| **强类型黑板** | DAG 节点间通过带 Schema 校验的共享黑板传递数据，杜绝幻觉 |
| **四阶段异步流转** | 长程异步操作必须经过触碰边界、平滑回复、深冻结、回调唤醒四个阶段 |
| **幂等防双花** | 所有写操作绑定 Task_ID + Card_Action_ID 作为全局唯一幂等键 |
